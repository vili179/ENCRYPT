## 1. Project Overview

ENCRYPT is a dual-layer steganography and encryption suite that combines XOR encryption with LSB (Least Significant Bit) steganography to hide encrypted messages within images. The application provides both a web interface (Express.js + EJS) and command-line tools.

### Technology Stack

| Layer | Technology | Version |
|-------|-----------|---------|
| Runtime | Node.js | >= 20 |
| Language | TypeScript | 5.7 |
| Web framework | Express.js | 4.21.0 |
| Template engine | EJS | 3.1.10 |
| Image processing | Sharp | 0.33.5 |
| File uploads | Multer | 1.4.5-lts.1 |
| Security headers | Helmet | 8.0.0 |
| Compression | compression | 1.7.5 |
| Client bundler | esbuild | 0.24.0 |
| Dev server | tsx | 4.19.0 |

### 3.3 `src/services/crypto.ts` — Encryption Service

**Purpose**: Provides symmetric XOR encryption and decryption for message content.

**Exports**:

#### `function xorCrypt(text: string, password: string): string`

| Parameter | Type | Description |
|-----------|------|-------------|
| `text` | `string` | Plaintext to encrypt or ciphertext to decrypt |
| `password` | `string` | User-provided password for key derivation |
| **Returns** | `string` | Encrypted or decrypted result |

**Algorithm**:
1. Compute `keySum` by summing all character codes of the password (lines 2-5)
2. Derive single-byte key: `key = keySum % 256` (line 6)
3. XOR each character of `text` with `key` (lines 8-11)
4. Return the resulting string

**Characteristics**:
- Symmetric: the same function encrypts and decrypts
- Deterministic: same inputs always produce same output
- Key range: 0-255 (single byte)

### 3.4 `src/services/steganography.ts` — Steganography Service

**Purpose**: Implements LSB steganography for embedding encrypted messages into images and extracting them.

**Dependencies**:
- `sharp` — image processing
- `xorCrypt` from `./crypto.js` — encryption
- `textToBits`, `bitsToText` from `../utils/bits.js` — bit conversion
- `StegoResult` from `../types.js` — return type

**Exports**:

#### `async function hideMessage(imageBuffer: Buffer, secretText: string, password: string): Promise<StegoResult>`

| Parameter | Type | Description |
|-----------|------|-------------|
| `imageBuffer` | `Buffer` | Raw bytes of the carrier image |
| `secretText` | `string` | The secret message to hide |
| `password` | `string` | Encryption password |
| **Returns** | `Promise<StegoResult>` | Stego image buffer with metadata |
| **Throws** | `Error` | If message exceeds image capacity |

**Process** (lines 6-49):
1. Encrypt secret text via `xorCrypt()` (line 11)
2. Convert encrypted text to bit array via `textToBits()` (line 12)
3. Create bit array with 8 trailing zero bits as null terminator (line 14)
4. Decode image via Sharp: remove alpha, sRGB, raw pixels (lines 17-21)
5. Check capacity: `secretBits.length <= totalChannels` (line 25)
6. Embed bits into LSBs: `modified[i] = (pixels[i] & 0xFE) | secretBits[i]` (lines 31-34)
7. Re-encode as PNG via Sharp (lines 36-40)
8. Return `StegoResult` object (lines 42-48)

#### `async function revealMessage(imageBuffer: Buffer, password: string): Promise<string>`

| Parameter | Type | Description |
|-----------|------|-------------|
| `imageBuffer` | `Buffer` | Raw bytes of the stego image |
| `password` | `string` | Decryption password |
| **Returns** | `Promise<string>` | The decrypted secret message |
| **Throws** | `Error` | If no message found or password incorrect |

**Process** (lines 51-96):
1. Decode image via Sharp: remove alpha, sRGB, raw pixels (lines 55-59)
2. Extract LSBs: `bits[i] = pixels[i] & 1` for all channels (lines 62-65)
3. Convert bits to text via `bitsToText()` (line 67)
4. Check for empty extraction (lines 69-71)
5. Decrypt via `xorCrypt()` (line 73)
6. Check for empty decryption (lines 75-79)
7. Validate printable character ratio >= 70% (lines 81-93)
8. Return decrypted message (line 95)

### 3.5 `src/utils/bits.ts` — Bit Conversion Utilities

**Purpose**: Converts between text strings and individual bit arrays for the steganography layer.

**Exports**:

#### `function textToBits(text: string): Uint8Array`

| Parameter | Type | Description |
|-----------|------|-------------|
| `text` | `string` | Text to convert to bits |
| **Returns** | `Uint8Array` | Array of individual bits (0 or 1), length = text.length * 8 |

**Algorithm** (lines 1-10):
- For each character, extract 8 bits in MSB-first order
- Bit extraction: `(charCode >> (7 - bitIndex)) & 1`
- Output size: exactly `text.length * 8` bits

#### `function bitsToText(bits: Uint8Array): string`

| Parameter | Type | Description |
|-----------|------|-------------|
| `bits` | `Uint8Array` | Array of individual bits (0 or 1) |
| **Returns** | `string` | Reconstructed text string |

**Algorithm** (lines 12-23):
- Process bits in groups of 8
- Byte reconstruction: `byte = (byte << 1) | bits[i + b]`
- Null termination: stops at first byte equal to 0
- Guard: `i + 7 < bits.length` prevents reading past the end

### 3.6 `src/utils/validation.ts` — File Validation

**Purpose**: Validates uploaded files against an allowed extension whitelist.

**Constants**:

#### `ALLOWED_EXTENSIONS: Set<string>`

Contains: `"png"`, `"jpg"`, `"jpeg"`, `"heic"`, `"heif"`

**Exports**:

#### `function isAllowedFile(filename: string): boolean`

| Parameter | Type | Description |
|-----------|------|-------------|
| `filename` | `string` | The original filename to validate |
| **Returns** | `boolean` | `true` if extension is allowed |

**Algorithm** (lines 3-7):
1. Find the last dot in the filename (`lastIndexOf(".")`)
2. If no dot found, return `false`
3. Extract extension (substring after last dot), convert to lowercase
4. Check if extension exists in `ALLOWED_EXTENSIONS` set

| Method | Path | Handler | Description |
|--------|------|---------|-------------|
| GET | `/hide` | Renders `hide.ejs` | Empty form with `error: null` |
| POST | `/hide` | Processes upload | Multipart form with `image`, `secret`, `password` |

**POST handler flow** (lines 17-51):
1. Extract `file`, `secret`, `password` from request
2. Validate file exists and passes `isAllowedFile()` → error if invalid
3. Validate `secret` and `password` are non-empty → error if missing
4. Call `hideMessage(file.buffer, secret, password)`
5. Set response headers: `Content-Type: image/png`, `Content-Disposition: attachment; filename="hidden_image.png"`
6. Send the stego image buffer
7. On error: render `hide.ejs` with error message

## 4. CLI Tools (`cli/`)

### 4.1 `cli/hide.ts` — CLI Hide Command

**Purpose**: Command-line tool for embedding a message into an image.

**Usage**:
```
npx tsx cli/hide.ts <input> <message> <password> <output>
```

| Argument | Description |
|----------|-------------|
| `input` | Path to the carrier image file |
| `message` | The secret text to embed |
| `password` | Encryption password |
| `output` | Path for the output stego image (PNG) |

**Process** (lines 1-18):
1. Parse command-line arguments (4 required)
2. Read input image into `Buffer` via `fs/promises.readFile()`
3. Call `hideMessage(imageBuffer, message, password)`
4. Write result buffer to output path via `writeFile()`
5. Print summary: bits used, image dimensions, output path

---

### 4.2 `cli/extract.ts` — CLI Extract Command

**Purpose**: Command-line tool for extracting a hidden message from an image.

**Usage**:
```
npx tsx cli/extract.ts <image> <password>
```

| Argument | Description |
|----------|-------------|
| `image` | Path to the stego image file |
| `password` | Decryption password |

**Process** (lines 1-13):
1. Parse command-line arguments (2 required)
2. Read image into `Buffer` via `fs/promises.readFile()`
3. Call `revealMessage(imageBuffer, password)`
4. Write decrypted message to stdout

### 5.2 `public/ts/hide.ts` — Hide Page Logic

**Purpose**: Manages the hide form interactions including file selection, character counting, password strength, and form validation.

**Behaviors**:

#### File Drop Zone (line 11)
- Calls `setupFileDrop("hideDrop", "hideFile", "ready")`

#### Character Counter (lines 13-22)
- Listens to `input` events on `#hideSecret` textarea
- Updates `#charCount` with current character count

#### Password Strength Meter (lines 24-42)
- Listens to `input` events on `#hidePassword`
- Computes strength score (0-4) based on:
  - `+1` if length >= 6
  - `+1` if length >= 10
  - `+1` if contains both uppercase and lowercase
  - `+1` if contains digits or special characters (`!@#$%^&*`)
- Colors segments `#s1` through `#s4`:
  - Score 1: `#ff3366` (red)
  - Score 2: `#ffaa33` (orange)
  - Score 3: `#00ffaa` (green)
  - Score 4: `#00ffcc` (cyan)

#### Form Validation (lines 44-67)
- Prevents submission if:
  - No file selected → toast: "Please upload a carrier image"
  - Empty message → toast: "Please enter a secret message"
  - Empty password → toast: "Please set a password"
- Shows `#hideLoading` spinner on valid submission

### 5.3 `public/ts/extract.ts` — Extract Page Logic

**Purpose**: Manages the extract form interactions including file selection, form validation, result display with typewriter effect, and clipboard copy.

**Behaviors**:

#### File Drop Zone (line 11)
- Calls `setupFileDrop("exDrop", "exFile", "ready for extraction")`

#### Form Validation (lines 13-32)
- Prevents submission if:
  - No file selected → toast: "Please select a stego image"
  - Empty password → toast: "Please enter the decryption password"
- Shows `#extractLoading` spinner on valid submission

#### Copy to Clipboard (lines 34-40)
- Click handler on `#copyBtn`
- Copies `#resultText` content via `navigator.clipboard.writeText()`
- Shows success toast on completion

#### Clear Result (lines 42-45)
- Click handler on `#clearBtn`
- Hides `.result-box` by setting `display: none`

#### Typewriter Effect (lines 47-61)
- Activates when `#resultText` has content (server-rendered result)
- Captures full text, clears element, then types character by character
- Speed calculation: `Math.max(15, Math.min(50, 500 / fullText.length))`
  - Short messages: slower (up to 50ms per char)
  - Long messages: faster (down to 15ms per char)
- Uses `setInterval` cleared when complete

### 9.3 Client-Side Dependency Model

Client scripts have no module imports. Cross-file communication uses the global `window` object:

```
common.ts
  ├── exposes: window.showToast()
  └── exposes: window.setupFileDrop()

hide.ts
  ├── calls: setupFileDrop()    [from window]
  └── calls: showToast()        [from window]

extract.ts
  ├── calls: setupFileDrop()    [from window]
  └── calls: showToast()        [from window]
```

Script load order (enforced by `foot.ejs`):
1. `common.js` — always loaded first, registers globals
2. `{pageScript}.js` — loaded second, consumes globals
